# Research Findings: Python Ecosystem, Physics Engines, and Interactive Embeds

This document consolidates research findings to inform the technical decisions for the "Physical AI & Humanoid Robotics" textbook.

## Decision: Python Version and Core Libraries

*   **Decision**: Python 3.9+ will be the primary version used for all code examples and simulations. Key libraries for general-purpose AI and data manipulation will include `TensorFlow`, `PyTorch`, `Scikit-learn`, `NumPy`, and `Pandas`.
*   **Rationale**: Python 3.9+ offers a good balance of modern features, performance, and wide community support. `TensorFlow` and `PyTorch` are industry-standard deep learning frameworks with extensive documentation and educational resources, crucial for covering AI concepts. `Scikit-learn` provides essential tools for traditional machine learning. `NumPy` and `Pandas` are foundational for numerical computing and data analysis, respectively.
*   **Alternatives Considered**: Earlier Python versions were considered but rejected due to diminishing support and lack of modern features. Other niche AI/ML libraries were deemed less comprehensive for a foundational textbook.

## Decision: Physics Engines for Humanoid Robotics Simulations

*   **Decision**: `PyBullet` will be the primary physics engine for basic to intermediate humanoid robotics simulations due to its robust physics engine, ease of use, and Python integration. `MuJoCo` will be introduced for advanced dynamics and reinforcement learning examples.
*   **Rationale**: `PyBullet` is highly suitable for educational contexts, allowing for straightforward loading of robot models and simulation of physics. `MuJoCo` is excellent for complex, high-performance dynamics, making it valuable for advanced topics. Both offer strong Python APIs.
*   **Alternatives Considered**:
    *   `CoppeliaSim`: Highly versatile with multiple physics engines, but its GUI-centric approach might add complexity for pure Python code examples in a textbook.
    *   `Webots`: Open-source and cross-platform with good Python support, but `PyBullet` and `MuJoCo` offer more focused, programmatic control for physics.
    *   `Genesis (Genesis-World)`: A promising Python-native AI engine, but `PyBullet` and `MuJoCo` have more established educational ecosystems.
    *   `NVIDIA Isaac Sim` / `Newton`: Powerful, GPU-accelerated platforms, but may have higher setup complexity and resource requirements for a general textbook.

## Decision: Interactive Embedding Solutions for Docusaurus

*   **Decision**: The `@docusaurus/theme-live-codeblock` plugin will be used for embedding interactive code editors for JavaScript/React examples. For Python, the `docusaurus-live-brython` plugin will be investigated for live Python code execution. MDX and custom React components will be leveraged for more complex interactive elements.
*   **Rationale**: The official Docusaurus live codeblock provides a seamless and integrated experience for interactive JavaScript/React code. `docusaurus-live-brython` offers a potential solution for live Python, which is crucial for the textbook's content. MDX and custom React components offer the flexibility needed for various interactive elements like quizzes and custom simulations.
*   **Alternatives Considered**:
    *   External platforms like HowdyGo, StackBlitz, and CodeSandbox: While powerful, embedding external platforms might introduce external dependencies, require more complex integration, or affect the self-contained nature of the textbook. They can be considered for advanced examples or supplementary resources, but an integrated solution is preferred for core content.
    *   Manual iframe embedding: Offers flexibility but lacks the integrated functionality and developer experience of Docusaurus plugins.
